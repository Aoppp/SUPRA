# DoLPHiN 文献总结

**论文标题**：DoLPHiN: a proposed architecture and technical implementation of a database of structural and functional properties of natural polysaccharides with health benefits
**期刊**：Food Chemistry 495 (2025) 146534
**作者单位**：武汉理工大学 + 华中科技大学同济医学院
**上线时间**：2025年9月29日

---

## 1. 核心定位

DoLPHiN（Database of Long-chain Polysaccharide for Health in Nature）是一个天然多糖结构与健康功能关系的在线数据库，免费开放，无需登录。
- **网址**：http://www.dolphindatabase.net/
- **当前数据量**：5085 条多糖条目
- **文献来源**：PubMed、Web of Science、Google Scholar 超过 23,000 篇文献，人工提取

---

## 2. 每条数据的字段结构

| 字段 | 说明 |
|------|------|
| 提取来源物种 | 物种名称 + TaxID |
| 多糖名称 | 原文作者命名 |
| 单体组成及百分比 | 26 种单体类型，按 KEGG Monosaccharide Codes 分类 |
| 分子量 | 统一为 kDa 单位 |
| 主链结构 | 糖苷键连接方式与序列 |
| 侧链结构 | 如有分支 |
| 糖苷键类型 | α/β 构型 + 连接位点 |
| 健康功能 | 抗氧化/免疫调节/抗肿瘤/抗糖尿病等 10 类 |
| 肠道微生物调节 | 是否报告了对肠道菌群的影响 |
| 是否含 NMR 信息 | Yes/Unknown |
| 原始文献链接 | DOI |

## 4. 核心功能模块

### 4.1 多维度检索
用户可按以下条件组合筛选多糖：
- 提取来源物种
- 多糖名称
- 单体组成百分比（如"葡萄糖含量 >20%"）
- 分子量范围（如 10-50 kDa）
- 糖苷键类型（如 β-Glcp-(1→4)）
- 健康功能（如 immunoregulation）

### 4.2 详情浏览
点击任一结果后展示完整信息，包含三个 tab：
- **Composition** — 单体组成
- **Structure** — 主链/侧链结构及糖苷键
- **Functions** — 健康功能及 ICD-11 相关疾病

### 4.3 关联图谱（Correlation Map）
- 每个物种详情页嵌入 Neo4j 可视化
- 展示"物种 → 多糖 → 单体/糖苷键 → 生理功能"三方关系网络，可点击交互

### 4.4 数据上传与审核
- 提供在线提交工具，支持文本或 GlycanBuilder 结构图
- **两轮审核**：学生团队初审（完整性/一致性）→ 两位独立专家复审（结构准确性/文献对齐），通过后入库

### 4.5 关联性分析
- 卡方独立性检验：单体组合 × 20 种健康功能
- 糖苷键类型 × 健康功能
- 显著性阈值 p ≤ 0.05
- 主要发现：葡萄糖是抗氧化活性的核心结构基础；β-Glcp-(1→6) 与免疫调节显著相关；α-Glcp-(1→6) 与肠道菌群调节显著相关

## 5. 数据库统计快照

| 维度 | 数据 |
|------|------|
| 最常见单体 | 葡萄糖（80% 条目含）> 半乳糖 > 甘露糖 > 阿拉伯糖 |
| 最常见组合 | Glc / Glc+Man+Gal / Glc+Gal |
| 最常见糖苷键 | α-Glcp-(1→4), β-Galp-(1→4), α-Glcp-(1→6), β-Glcp-(1→6) |
| 最常见功能 | 抗氧化（2204条）> 免疫调节（1532条）> 抗肿瘤（821条） |
| 含肠道菌群数据 | 仅 81 条 |

## 6. 局限性与未来方向

1. 各文献的实验方法不统一，数据客观性受影响
2. 多数研究仅关注单一方面功能，缺乏对同一多糖的持续深入研究
3. 肠道微生物相关数据极为稀缺（仅 81/5085）
4. 抗氧化以外（如抗肿瘤、免疫调节）的评估方法差异大，结果不可直接横向对比
5. 计划定期更新，并呼吁用户协助提交遗漏数据

## 7. 对你开发同类网站的参考价值

如果你要复刻这个网站，核心模块拆分如下：

### 最小可行产品（MVP）应包含：
1. **数据模型**：Species → Polysaccharide → Monomer/Linkage/Function 的三实体 ER 模型
2. **多条件检索**：至少支持物种、名称、单体、分子量范围、功能五个维度
3. **详情页**：Tab 切换展示结构/组成/功能
4. **数据管理后台**：录入 + 审核工作流

### 可选的增强功能（DoLPHiN 有但你起步可跳过）：
- Neo4j 图数据库 + 关联图谱可视化（需要额外服务器和开发量）
- 卡方检验等统计分析模块
- 用户上传 + 审核工单流

### 技术选型建议：
- 若你熟悉 Python → 直接用 Django + MySQL 是最短路径（和原论文一致）
- 若你想更快出原型 → Supabase（PostgreSQL + 自带 API）+ Next.js 前端，省掉后端 CRUD 开发
- 图数据库可暂缓——Elasticsearch 的聚合查询也能实现类似的多维筛选
